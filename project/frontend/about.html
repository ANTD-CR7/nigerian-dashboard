About This System

Design and Implementation of a Web-Based Financial Data Aggregation and Analytics System Using Nigerian Public Economic Data - 11,022 records, 115 indicators, 2020-2026

This system aggregates 11,022 observations across 115 Nigerian economic indicators from the CBN, NBS and World Bank into a single interactive web dashboard - making public economic data accessible to all Nigerians. Coverage spans GDP sectoral breakdown (57 sectors, 1981-2024), multi-currency exchange rates (12 currencies, 2020-2026), FX reserves, CBN balance sheet, NFEM daily rates and the original four core macro indicators.

Project Identity

This project is not a static archive of old figures. It is a mini financial data platform that aggregates public economic data, normalizes it into a common time-series format, stores it in an appendable database, and exposes analytics through API endpoints and dashboard controls.


The 2020-2024 period is used as a controlled case study for consistency, validation, and reproducibility. The implementation supports future data through demo-safe manual API validation and CSV ingestion, with real database writes gated by a server-side configuration flag.

System Overview

This system aggregates 11,022 observations across 115 indicator IDs from three official public data sources covering January 2020 to April 2026. It presents the data through an interactive web dashboard with multi-series charts, cross-indicator analytics, a public REST API, and summary tables that any user can access through a standard web browser.


This project was developed as a Final Year Project at Caleb University, Imota, Lagos, in the Department of Computer Science. Its contribution is a free, open-source, and academically documented implementation built on a teachable technology stack that Nigerian computer science students can study, understand, and extend.

Data Collection Method

All data was collected by manually downloading publicly available Excel and CSV files from the official websites of the three source institutions. The data was then processed using Python and Pandas following the ETL (Extract, Transform, Load) framework:


Extract: Excel and CSV files downloaded from CBN, NBS, and World Bank websites.


Transform: Python Pandas used to extract relevant columns, standardise date formats to ISO 8601 (YYYY-MM-DD), convert strings to numeric values, and filter for the 2020-2024 date range. For exchange rate, daily CBN values were averaged to monthly. For GDP, the "GDP at 2010 constant price" row from the "real gdp growth rate %" sheet was extracted from each quarterly NBS report.


Load: Cleaned data inserted into the Supabase PostgreSQL database using the Supabase Python client library.
